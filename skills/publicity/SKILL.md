# Publicity

## Principles

- Reason per work item or project: what specific result, artifact, or knowledge
  deserves more visibility, and for whom?
- Start from substance. Improve access to evidence before amplifying claims.
- Use `custom-search` first, including existing posts/content, to avoid
  duplicating prior messaging.
- Do not take publicity actions from this skill. Output suggestions for the
  user to approve, reject, or revise; public actions need the normal approval
  path.

## Actions To Consider

- Upload or expose artifacts: GitHub release, repo artifact, website page,
  benchmark result, plot, dataset, model, demo, or documentation.
- Publish research: arXiv/preprint, journal or workshop submission, DOI/Zenodo,
  citation block, or project page.
- Share updates: X, LinkedIn, website post, README highlight, newsletter, or
  relevant community post.
- Contact people: email maintainers, researchers, users, collaborators, or
  communities who would benefit from the result.
- Share knowledge: turn useful vault knowledge or project evidence into a
  public note, explainer, or resource when appropriate.

## Output

Return a short prioritized list: action, audience, evidence/artifact to link,
risk or approval needed, and expected value.

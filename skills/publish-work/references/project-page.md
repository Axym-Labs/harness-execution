# Paper to project page

Keep the paper head's internal order so the page can reuse the source with
minimal rearrangement:

1. Title, authors, affiliations and city
2. Lead figure or immediately understandable interactive element, when useful
3. Abstract (once)
4. Important links: paper, code, demo, data/model, documentation, slides
5. Condensed motivation and method
6. Main results
7. Ready-to-copy BibTeX
8. Further detail, examples, or a more involved interactive element

An interactive element may instead follow the abstract when it needs the
abstract to be understood. Extend the important-link row only when another
artifact is useful. Condense the paper; do not turn the project page into a
second, differently ordered paper.
